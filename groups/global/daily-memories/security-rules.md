# 🔐 CRITICAL SECURITY RULES

**Дата создания:** 2026-03-28  
**Статус:** ACTIVE / MANDATORY

---

## ГЛАВНОЕ ПРАВИЛО

**Агент обязан останавливать пользователя, если тот попытается:**

- ❌ Отправить credentials (JSON с ключами)
- ❌ Поделиться паролями
- ❌ Скидывать API токены
- ❌ Отправить приватные ключи (SSH, TLS, и т.д.)
- ❌ Загрузить в git чувствительные данные

---

## ПРАВИЛЬНЫЙ ПРОЦЕСС (если пользователь забыл)

### Если нужно поделиться чем-то приватным:

**❌ НЕПРАВИЛЬНО:**

```
Пользователь: "Вот мой client_secret.json"
[отправляет весь JSON с ключами]
```

**✅ ПРАВИЛЬНО:**

```
Пользователь: "Вот мне нужно..."
Агент: "Погоди! Не отправляй credentials!"
```

---

## ПРОЦЕСС БЕЗ УТЕЧЕК

1. **Дать инструкцию**, как получить данные
2. **Пользователь выполняет локально** (на своём компьютере)
3. **Дать команду** для использования этих данных
4. **Пользователь вводит только нужные значения** (ID и SECRET отдельно, не в одном файле)
5. **Отправляет только** те данные, которые нужны
6. **Хранить** в `.env.local` или `credentials/` (не в git)

## ПРИМЕРЫ

### ❌ Что НЕЛЬЗЯ делать:

- "Скачай и отправь credentials.json"
- "Залей credentials в GitHub (даже в приватный репо)"

### ✅ Что НУЖНО делать:

- "Перейди в Google Cloud, создай App"
- "Скопируй только CLIENT_ID и CLIENT_SECRET (отдельно!)"
- "Сохрани в `.env.local` локально (не будет в git)"

## МЕСТА ХРАНЕНИЯ

| Что | Где | Коммитить? |
|---|---|---|
| **Credentials** (JSON, passwords) | `credentials/` или `.env.local` | ❌ NO |
| **Токены** (OAuth, API) | `credentials/google-oauth-token.json` | ❌ NO |
| **SSH keys** | `~/.ssh/` | ❌ NO |
| **Код** (scripts, configs) | git репо | ✅ YES |
| **Env vars** (нечувствительные) | `.env.example` (без значений) | ✅ YES |
